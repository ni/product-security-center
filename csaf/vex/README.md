# Vulnerability Exploitability Exchange (VEX) Repository
This repository facilitate the sharing of VEX disclosures related to third-party Common Vulnerabilities and Exposures (CVEs) affecting NI products.

## What is VEX?
From [When to Issue a VEX](https://www.cisa.gov/sites/default/files/2023-11/When-to-Issue-a-VEX-508c.pdf):
Vulnerability Exploitability eXchange or VEX is a human-readable and machine-comprehensible document that allows a software supplier or other party to assert the status of specific vulnerabilities in a particular product.  Issuing VEX information allows developers, suppliers, and others to provide information about a vulnerability, regardless of whether software is affected by a specific vulnerability. This information allows downstream users to make their own assessments of the risks associated with a vulnerability.  

Unlike general vulnerability disclosures, VEX focuses on whether a vulnerability in a component is actually exploitable in its specific context.

## CSAF VEX Profile
The files in this repository follow the CSAF VEX profile.  Specifically, the */document/category* is *csaf_vex*

## When does NI issue VEX
VEX statements are a part of our ongoing commitment to transparency around security and giving users access to timely and accurate information when possible.  However, this is a work in progress.  At present, NI primarily issues public VEX statements in this repo when there are third-party CVEs of significant public or user attention.
